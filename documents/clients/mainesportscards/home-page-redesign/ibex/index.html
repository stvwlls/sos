Welcome to Ibex

What is Ibex?

Ibex is a minimal frontend SCSS framework for creating websites.

This Style Guide illustrates the scope of what Ibex sets out to do, which is:

  • Provide a quick and easy way for an iBec Developer to generate basic page layouts.
  • Make the Typography process easier for multiple devices using fluid type.
  • Provide basic styles for the most common elements.
  • Get out of your way so you can do great work.

This Style Guide also serves as a style guide, which will update itself as you code. The only manual step you need to take is to update the color swatches for your website's colors.

Why should I use this? I like doing everything on my own.

We all do, the problem comes in when everyone has their own custom methods for doing things that others may not understand. Since you work with other developers we need a common basis that we all work from. When someone starts a new project or inherits an old one, the structure of the project and it's foundational CSS should be familiar to everyone.

The file structure of Ibex is one of the most important parts, and it's flexible. You can make your own additions to it, you can use or not use some of the boilerplate features - it's up to you.

File Structure:

The file structure of Ibex is set up using a modular CSS approach. There is a main styles.scss file that grabs _index.scss files from each of the subdirectories. This enables you to add new components on the fly without having to stop and start Gulp.

Below is the base directory & file structure for Ibex:

/abstracts
  - _index.scss
  - _mixins.scss
  - _variables.scss

/base
  - _index.scss
  - _reset.scss
  - _typography.scss

/components
  - _buttons.scss
  - _forms.scss
  - _images.scss
  - _index.scss
  - _tables.scss

/layout
  - _footer.scss
  - _grid.scss
  - _header.scss
  - _index.scss
  - _navigation.scss

/pages -- Add any new pages you need to style specifically here.
  - _home.scss
  - _index.scss

/vendors -- For any CSS from JS plugins.
  - _index.scss
